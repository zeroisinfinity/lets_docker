Here’s the **full reference** for `docker ps --filter` (or `docker container ls --filter`) with all known filters and how you can **combine** them logically.

---

## **All `docker ps` Filters**

You can combine **multiple filters** simply by repeating `-f` or `--filter`:

```bash
docker ps -a -f status=exited -f name=mycontainer
```

---

### **1. `id=`**

Match by **full or partial container ID**.

```bash
docker ps -a -f id=686a0e9c65ae
```

* Matches: Only that exact container.
* Combine: Works with all others (`status`, `before`, etc.).

---

### **2. `name=`**

Match by container name (substring match).

```bash
docker ps -a -f name=web
```

* Matches: Containers with `web` anywhere in the name.
* Combine: Often with `status` or `before`.

---

### **3. `status=`**

Match by container status:

* `created` — Created but never started.
* `restarting` — Restarting after a crash.
* `running` — Currently running.
* `removing` — Being removed.
* `paused` — Paused state.
* `exited` — Finished running.
* `dead` — Failed, unrecoverable state.

```bash
docker ps -a -f status=running
```

---

### **4. `ancestor=`**

Match by **image name**, **tag**, or **image ID**.

```bash
docker ps -a -f ancestor=ubuntu:25.04
```

* Combine: Perfect for finding all containers from the same image.

---

### **5. `before=`**

Match containers **created before** a specific container (name or ID).

```bash
docker ps -a -f before=focused_borg
```

---

### **6. `since=`**

Match containers **created after** a specific container.

```bash
docker ps -a -f since=angry_nightingale
```

---

### **7. `volume=`**

Match containers with a specific volume **mount path**.

```bash
docker ps -a -f volume=/data
```

---

### **8. `network=`**

Match containers connected to a specific network.

```bash
docker ps -a -f network=bridge
```

---

### **9. `label=`**

Match containers by Docker label.

```bash
docker ps -a -f label=maintainer=John
```

* Just key: `-f label=env`
* Key=Value: `-f label=env=prod`

---

### **10. `health=`**

Match by health check status:

* `starting`
* `healthy`
* `unhealthy`
* `none` (no healthcheck defined)

```bash
docker ps -a -f health=healthy
```

---

## **Combining Filters**

Just repeat the `-f` flag:

```bash
docker ps -a \
  -f status=exited \
  -f ancestor=ubuntu:25.04 \
  -f before=focused_borg
```

This is **AND logic** — a container must match **all** filters to appear.

---

## **Pro Tip — List Only IDs**

When scripting:

```bash
docker ps -aq -f status=exited -f ancestor=ubuntu:25.04
```

Alright — --format in docker ps is where you stop being at the mercy of Docker’s default table and start shaping the output exactly the way you want.

It uses Go templates (the same templating engine Docker uses internally).

Basic Usage
docker ps --format "{{.ID}}: {{.Image}}"


Outputs something like:

c8c5dd363b03: ubuntu:25.04
7083b0630e4b: ubuntu:25.04

Common Built-in Placeholders
Placeholder	Meaning
{{.ID}}	Short container ID
{{.Image}}	Image name used
{{.Command}}	Command run in container
{{.CreatedAt}}	Creation time
{{.RunningFor}}	Human-readable uptime
{{.Ports}}	Published ports
{{.Status}}	Current container status
{{.Size}}	Size (if using --size)
{{.Names}}	Container names
{{.Labels}}	All labels as a map
{{.Mounts}}	Mount points
Table Format

If you want headers:
```bash
docker ps --format "table {{.ID}}\t{{.Image}}\t{{.Status}}"
```
